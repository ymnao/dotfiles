# claims — 主張と証拠のギャップ

実セッションで最も指摘件数が多かった観点。証拠より一段〜二段強い言い回しを
検出し、証拠の強さに釣り合う表現への弱め方を必ず提案する(「弱めろ」だけの
指摘は出さない。before → after を書く)。

## 1. 過大表現のパターン(検出語 → 弱め方の実例)

| 過大表現 | 証拠がそこまで無い場合の弱め方 | なぜ |
|----------|------------------------------|------|
| 上限/境界を確定 | 探索した範囲では / 急増域を確認 | 2 点比較で言えるのは区間の存在まで |
| 相転移 | 閾値様急増 | 統計力学用語。細かい走査・有限サイズ分析なしでは過大 |
| 網羅的に探索 | 探索した系統を列挙して限定(「2 系統を探索」) | 実際の探索範囲に合わせる |
| 残差の主因 | 有力な要因の一つ | 未検討要因が残る限り主因特定は不可 |
| 機構限界の実証 | 有力候補とする | |
| 原理的に不可能 / 再現できない | 今回探索した範囲では / 再現困難 | |
| (シミュレーション研究での)因果的主張 | 本モデル内の介入効果 | モデル内効果と外的妥当性の区別 |

英文の hedging 強度階層(和文の断定表現にも同じ階層を適用する):

- 強: prove / demonstrate / establish / confirm(実証・確定・立証)
- 中: suggest / indicate / appear to(示唆・示す・とみられる)
- 弱: may / could / might / seem(可能性がある・かもしれない)

主張の強度が証拠の強度(比較点数、探索範囲、統計的裏付け、再現回数)を超えて
いる箇所をすべて挙げ、一段〜二段下の語への置換を提案する。逆に、十分な証拠が
あるのに過剰に弱めている箇所(hedging の重ね掛け "may possibly suggest")も指摘
してよい。

## 2. 自己矛盾の検出

結論・帰属の言明を、本文中の**自分の肯定的結果**(図表・数値・別節の記述)と
横断的に突き合わせる。実例: 「volatility clustering の内生的生成源の不在」と
書きながら、直前の図で弱い正の自己相関(=生成源の存在を示す肯定的結果)を報告
していた → 「持続的な時変分散を生む内生的機構の不足」に修正。

チェック手順: 否定形・不在形の言明(「〜は見られない」「〜を欠く」)を抽出し、
それぞれについて矛盾する図表・数値が原稿内に無いか検索する。

## 3. スコープの整合

- abstract / conclusion の主張が本文の結果でカバーされているか(abstract は
  本文より強くなりがち)
- limitations に書いた制約と、本文中の断定表現が矛盾していないか
- 「今後の課題」に回した項目を、別の節で既達成のように書いていないか
