# format — 書式・組版

LaTeX 原稿を第一級として扱う。学会・投稿先のフォーマット規定(スタイルファイル
の説明書、著者ガイド、見本 PDF)がリポジトリ内にあれば最初に読むこと。

## 1. 学会フォーマット準拠(LaTeX)

- 所属マーク・所属注の位置と記法(見本 PDF と突き合わせる。実例: 所属注の
  カラム位置・「†」vs「1)」が見本と不一致だった)
- 脚注罫線、段組、フォントサイズ等のテンプレート逸脱
- 文献リスト: 引用順ソート(または投稿先指定の順)、thebibliography のラベル幅、
  書誌フォーマットの統一
- 数式書体: 統計量・変数はイタリック($U$, $p$, $n$)、単位・演算子名は立体
  (ISO 80000-2)

## 2. 図表

- **キャプションの自立性**: 図表 + キャプションだけで読めるか(略語・条件名が
  キャプション内で説明されているか)
- 内部 ID・英語タイトルの露出: 実装の run ID(例: baseline_hetero_full_...)や
  作図スクリプトの英語タイトルが図中に残っていないか
- 凡例の語の誤読リスク: 実例: シミュレーション結果の系列に "empirical"
  (実データと誤読される)→ "simulated" に修正
- 図の描画範囲と本文の判定範囲の齟齬(本文は区間 [a,b] で判定と書いているのに
  図は別範囲を表示)
- プール分布の図と集約統計(seed 別平均等)の報告値の区別がキャプションで
  明示されているか
- TikZ 等の描画: ラベルの重なり、線の交差による誤読

## 3. ページ制約

- 制約(ページ数・段数)に対する現状を必ずリビルドで確認する。「たぶん収まる」
  で完了と言わない
- 圧縮の定石と、削った内容の対応先記録は SKILL.md 手順 5 に従う。format 固有の
  追加観点: 段抜き表のセル圧縮は両カラム分効くので優先度が高い。図の縮小は
  キャプション可読性とトレードオフになるため最後の手段にする

## 4. 和文組版

- 句読点(、。)の統一、全角/半角の使い分け(英数字は半角、括弧の全半角統一)
- 波ダッシュ/全角チルダ、ハイフン/マイナス/ダッシュの使い分け
- 和欧混植のスペース(自動処理に任せるか手動スペースか、どちらかに統一)

## 5. Markdown 原稿

- 見出しレベルの整合、リンク切れ、画像パス
- pandoc 等で変換する運用なら変換後の体裁も確認対象(変換コマンドは
  リポジトリの設定から検出)
