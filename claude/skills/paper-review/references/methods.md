# methods — 統計・方法論

統計の指摘は必ず実験設計(データの生成過程)まで遡って確認する。数式・手法名の
表面だけで判定しない。数値の実データ突合そのものは facts 観点の担当(重複して
見つけた場合はどちらが報告してもよい)。

## 1. 検定と実験設計の対応

- **独立 vs 対応あり**: 条件間で乱数 seed・被験者・データセットを共有している
  (common random numbers 等)なら対応あり設計 → 対応あり検定(Wilcoxon 符号付
  順位、対応 t)。独立 2 群検定(Mann-Whitney U、独立 t)は不整合。設定ファイル
  や実験手順書で seed の共有を確認する
- パラメトリック検定の前提(正規性・等分散)が確認されているか、または
  ノンパラが選ばれた理由が説明できるか

## 2. 擬似反復(pseudoreplication)

- 反復単位は何か(seed / 被験者 / セッション)。下位サンプルをプールした n を
  独立標本として検定していないか
- 点推定・図は反復単位別の集約(seed 別平均等)+ CI で報告しているか。プール
  分布の図と集約統計の報告値を混同していないか

## 3. 報告の規範(ASA statement 準拠)

- p 値のみの報告は不可。効果量と 95% CI を併記する
- 統計的有意と実質的重要性の混同(「有意 = 効果が大きい」)がないか。
  "significant" の日常語的用法の混入も検出する
- 多重比較: 複数検定があれば補正(Bonferroni / FDR)の有無・手法・選択理由が
  明示されているか
- n・自由度・検定名・片側/両側が明示されているか
- 非有意の結果も報告されているか(selective reporting の兆候)

## 4. 分布・数値基準の前提

- 分布の理論的前提: 例 Student-t は ν≤4 で 4 次モーメントが存在しない
  (excess kurtosis = 6/(ν−4) は ν>4 でのみ定義)→ ν=3 等での kurtosis 比較には
  「極端な裾のストレス条件」等の注意書きが必須
- 数値基準の根拠: 「目標 ≥3」等の閾値に文書化された根拠(先行研究・規格・
  事前登録)があるか。なければ「暫定参考水準(絶対基準ではない)」への格下げを
  提案する

## 5. 仮説と検証の構造

- 仮説は方向性つきの検証可能な言明か。ラベルだけの H1/H2(「H1: 異質性の影響」)
  は不可 → 「H1: X が増えると Y が減る」形へ
- 定義的に自明な仮説(実装したルールがそのまま出力に出ることの確認)を
  「検証」と呼んでいないか → sanity check / verification に格下げ
- **verification(実装が仕様どおりか)と validation(モデルが対象を妥当に表現
  しているか)の区別**。構成要素の挙動確認を外部妥当性の証明のように書いて
  いないか
- 研究計画の頑健性(計画段階の原稿の場合): 基準モデルの置き換えでなく新旧両方
  での検証になっているか、調整用と最終検証用の乱数 seed が分離されているか、
  目標達成を前提にした計画表現(「〜を達成した後に」)になっていないか
