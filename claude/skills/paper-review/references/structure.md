# structure — 構成・論理

読者(その分野の査読者・指導教員)が最初に通読したときに迷う箇所・引っかかる
箇所を検出する。文単位でなく節・原稿全体を見る観点。

## 1. 概念の初出順と流れ

- 概念が定義・導入より先に使われていないか。実例: 「本ツール」が構築の宣言
  なしに突然登場 → 読者はどのツールか分からない
- 節ごとの中間結論が可視か(結果を並べただけで「だから何か」が節末にない)
- 図表と本文の参照関係: 全図表が本文から参照されているか、参照順と掲載順が
  一致しているか

## 2. 先行研究の扱い

- 中立性: 自説に有利な側の文献だけ引いていないか(悪化を報告した文献だけ /
  改善を報告した文献だけ、の偏り)。反対方向の代表的文献を挙げて位置づける
- 自分の貢献と先行研究の境界が明示されているか(「本研究の新規性は〜」が
  先行研究の記述と整合するか)

## 3. abstract(原稿に abstract がある場合)

構成: 背景・目的 → 手法 → 結果 → 含意、の要素が過不足なく入っているか。

- 主要な**定量的結果(数値)**が入っているか("significant improvement" だけで
  数値が無いのは不可)
- 本文で裏付けられていない結論・本文より強い主張が abstract に無いか
  (claims 観点と重複したら claims に譲る)
- abstract 単体で自己完結するか: 未定義の略語・内部用語が無いか
- 文献レビュー的な余談・冗長な背景が紛れていないか
- 時制(英文): 手法・結果は過去形、含意・結論は現在形
- 投稿先の語数制限・構造化指定に準拠しているか

## 4. 研究スコープと計画(計画段階・中間発表の原稿の場合)

- スコープの現実性: 残り期間に対して条件マトリクスが広すぎないか。優先順位
  つきの絞り込み(主要 2〜3 条件 + 感度分析)を提案する
- 研究計画の頑健性は methods 観点 §5 の担当(重複したら methods に譲る)
- 積み残し・限界が「今後の課題」として明示され、対応先(本編 / スライド /
  別稿)が書かれているか

## 5. 節構成の定石との照合

IMRaD(または分野の標準構成)からの逸脱が意図的か。Introduction の末尾に
貢献の列挙があるか、Discussion に limitations があるか、Conclusion が Results の
繰り返しになっていないか。
