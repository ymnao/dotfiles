# terms — 用語・定義・表現

「その分野の読者が読んで引っかかる語」を検出する。独自造語の判定には
WebSearch を使ってよい(分野での使用実績を確認する)。

## 1. 独自造語・直訳語の検出

分野で使われていない語を検出し、標準語への置換を提案する。判定に迷ったら
WebSearch で「その語 + 分野名」の使用実績を確認する。実例:

| 原稿の語 | 問題 | 標準語 |
|----------|------|--------|
| 静的レバー | policy lever の直訳 | 静的パラメータ |
| 板凍結 | 独自比喩 | 流動性枯渇 |
| 帰属分解 | 非標準 | 要因分解 |
| 群れ行動 | 過去リターン追随は trend following。herding は別概念 | トレンド追随 |
| 参照枠 | frame of reference の誤用 | 判定対象(文脈による) |

類似概念の混同(上の trend following vs herding のような「隣の概念」への
すり替わり)は単なる用語問題でなく主張の誤りになるため major 以上にする。

## 2. 定義と一貫性

- 略称: 初出で定義されているか。「以下、〜と呼ぶ」の定義と後続の使用が一致
  しているか。定義外略称・口語略称(「アルゴ」等のジャーゴン)の混入
- **操作的定義の欠如**: 分類語が排他的に定義されているか。実例:
  「アルゴリズム参加者」— 背景の FCN エージェントもアルゴリズムなのに区別が
  未定義 → 「背景参加者 / 介入対象」の操作的定義を提案
- 未定義の記号・変数: $r_t$ 等の数式記号は初出で定義されているか
- **コード内部識別子の露出**: activation、drop_ratio、run ID 等の実装名が本文・
  図にそのまま出ていないか → 論文用語に置換
- 表記統一: 和英混在の揺れ(超過尖度 vs excess kurtosis — 定義が 3 ずれる別量の
  混同にも注意)、同一概念の複数表記を全文検索で洗い出す

## 3. 英文特有(英文原稿のみ)

- **冠詞**: 可算/不可算 → 初出・不特定は a/an、既出・特定は the、総称は無冠詞
  複数。日本語話者の最頻出エラーなので全名詞句を機械的に見る
- **時制の使い分け**: 先行研究の一般的知見 = 現在形("X depends on Y")、
  個別研究の言及 = 過去形("Smith (2020) found")、分野の蓄積 = 現在完了形
  ("Studies have shown")、自分の手法・結果 = 過去形、結論・含意 = 現在形
- 態: 不必要な受動態の連続で主語が曖昧になっていないか(現代の主要誌は能動態
  + we を許容・推奨)。ジャーナル指定があればそれに従う
- 複合形容詞のハイフン: 名詞前の複合修飾語はハイフン("well-known method")、
  -ly 副詞 + 形容詞は不要("widely used method")
- LLM / 非母語話者の癖: "significantly" の非統計的濫用、過度な名詞化、
  紋切り型の導入("In recent years, ... has attracted much attention")、
  文長が均一で単調

## 4. 和文特有(和文原稿のみ)

- である体の統一、句読点(、。vs ,.)の統一
- 全角チルダ・波ダッシュ等の記号揺れ
- カタカナ語の長音符揺れ(サーバ vs サーバー)
